git

进入需要创建repository的本地的folder之后：

git init // 创建了git repository

然后在这个folder里增加你需要的东西，第一次需要上传的时候：

git add . // 把所有的都加进去 或者 git add <FILE_NAME>

接着必须写commit的comment：

git commit -m "这里随便你写什么留言，最好跟修改内容相关"

设置主branch：

git branch -M master // 主branch一般叫什么main啦或者master之类的，是所有版本的幕前人物

此时可以查看当前所在的branch：

git branch

因为是第一次上传，所以需要告诉本地的repo你云端的地址：

git remote add origin <此处是你github那个对应repo的地址>

继而就可以愉快push啦！

git push -u origin master // 此处是直接push到了master branch上

现在查看github的云端repo，就会发现已经上传上去了~

Later...在本地增删改了一些东西，要再次上传，最好切换分支！

新建一个branch（分支）：

git branch <blahblah分支名>

切换到那个分支：

git checkout <blahblah分支名>

此刻就在新的分支上了！不确定的话可以double check一下：

git branch

想要查看当前你修改了多少东西：

git status

把你修改的东西全部都加进去：

git add .

必写comment：

git commit -m "我创建了一个新的分支"

因为是在此分支上第一次push，所以还是要告诉云端一下本地分支应该对应云端的什么，即类似于在云端也新建一个branch来跟本地的这个branch对接：

git push —set-upstream origin <blahblah分支名>

当然如果记不了这么长的话，就直接打 git push，terminal会给提示的…然后复制粘贴就行了

云端有了新的branch，就可以选择跟master合并（merge）

这个可以直接在github上操作

之后在一个branch上如果不是第一次push了，那每次就只要：

git add .

git commit -m "我又修改了什么玩意儿"

git push

删除本地git repository的话，只要在那个repo的目录下输入：

rm -rf .git

如果本地版本过于落后于云端的master（一般出现在多人协作时），就需要把最新版本拉到本地：

git pull

当然还有什么git diff, git merge之类的……不过初级版就上面那些了……
